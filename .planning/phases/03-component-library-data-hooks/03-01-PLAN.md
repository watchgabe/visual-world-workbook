Create `src/hooks/useAutoSave.ts` implementing the unified auto-save hook per D-01, D-02, D-03, D-04.

**Exported interface (exact):**
```typescript
import type { ModuleSlug } from '@/types/database'

export interface UseAutoSaveOptions {
  moduleSlug: ModuleSlug
  fieldKey: string
  value: string
  onSaveSuccess?: () => void
}

export interface UseAutoSaveReturn {
  saveError: string | null     // null = no error; string = error message
  isFocused: boolean           // true while field has focus
  handleBlur: () => void       // attach to onBlur
  handleFocus: () => void      // attach to onFocus
  retry: () => void            // manual retry per D-04
}

export function useAutoSave(opts: UseAutoSaveOptions): UseAutoSaveReturn
```

**Implementation rules:**
1. `'use client'` directive at top
2. Use `useRef` (NOT `useState`) for `timerRef: ReturnType<typeof setTimeout> | null`, `controllerRef: AbortController | null`, `latestValueRef: string`, and `mountedRef: boolean` ŌĆö these must NOT cause re-renders
3. Use `useState` only for `saveError: string | null` and `isFocused: boolean` (these need to trigger re-renders for UI)
4. Get `user` from `useAuth()` ŌĆö if `user` is null, do not attempt saves (early return from doSave)
5. Get `refreshProgress` from `useProgress()` ŌĆö call after successful save per D-09

**Keep latestValueRef in sync (avoids stale closure pitfall per research Pitfall 2):**
```typescript
const latestValueRef = useRef(value)
useEffect(() => { latestValueRef.current = value }, [value])
```

**Cancel helper:**
```typescript
function cancelPending() {
  if (timerRef.current) clearTimeout(timerRef.current)
  if (controllerRef.current) controllerRef.current.abort()
}
```

**doSave function:**
- Abort any existing controller via `controllerRef.current?.abort()` per D-03
- Create new `AbortController`, store in `controllerRef`
- Call `createClient()` from `@/lib/supabase/client`
- Read value from `latestValueRef.current` (NOT from closure)
- Perform upsert: `supabase.from('blp_responses').upsert({ user_id: user.id, module_slug: opts.moduleSlug, responses: { [opts.fieldKey]: latestValueRef.current } }, { onConflict: 'user_id,module_slug' })`
- Add comment: `// Note: Caller should ideally pass full module responses to avoid partial overwrites. Phase 5 module pages use react-hook-form getValues() for this.`
- On success: check `mountedRef.current` before calling `setSaveError(null)`, then call `opts.onSaveSuccess?.()`, then call `refreshProgress(opts.moduleSlug)`
- On error: if `error.name === 'AbortError'` or `error.code === '20'`, silently return (expected cancellation). Otherwise check `mountedRef.current` before calling `setSaveError(error.message)`.

**Debounce effect (per D-02: 5s debounce OR blur, whichever first):**
```typescript
useEffect(() => {
  cancelPending()
  timerRef.current = setTimeout(doSave, 5000)
  return cancelPending  // cleanup on unmount or value change
}, [value])
```

**handleBlur:** Clear timer via `cancelPending()`, call `doSave()` immediately, set `isFocused` to false via `setIsFocused(false)`

**handleFocus:** Set `isFocused` to true via `setIsFocused(true)`, clear `saveError` (user re-engaging)

**retry:** Call `doSave()` directly (bypasses debounce timer)

**Mounted ref for cleanup (avoids React warning per Pitfall 3):**
```typescript
const mountedRef = useRef(true)
useEffect(() => {
  mountedRef.current = true
  return () => { mountedRef.current = false }
}, [])
```
