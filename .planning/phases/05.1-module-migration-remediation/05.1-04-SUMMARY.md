## Decisions Made
- Creator array persisted as JSON string in single blp_responses field (D-07): avoids dynamic field explosion, simple to serialize/deserialize
- `useEffect([creators, setValue])` bridges local React state to react-hook-form: auto-save watch fires naturally on any creator change without custom save logic
- `CreatorCard` extracted as sub-component within the same file: keeps creator array state in parent component, clean props interface (onUpdate/onRemove/onAnalyze callbacks)
- Per-card error state via `Record<string, string>` keyed by creator ID: independent error display per creator, clears on re-analyze or remove
