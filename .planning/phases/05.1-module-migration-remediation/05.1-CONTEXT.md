# Phase 05.1: Module Migration Remediation - Context

**Gathered:** 2026-04-02
**Status:** Ready for planning

<domain>
## Phase Boundary

Restore all missing features, content, and AI integrations from the old HTML modules that were not migrated in Phase 5. The user-facing experience must be identical to the old app — we are improving the codebase, not the content or UX. Every feature, section, button, and interaction from the old HTML must exist in the new React app.

</domain>

<decisions>
## Implementation Decisions

### Guiding principle
- **D-01:** Exact 1:1 with old app. The codebase improves, the user-facing experience stays identical. When in doubt, match the old HTML behavior.

### AI generation features
- **D-02:** AI-generated text populates the target field directly (setValue). User can edit after. Matches old app's behavior exactly.
- **D-03:** AI prompts are migrated 1:1 from the old edge function — no prompt improvements. The client sends context fields, the edge function has the prompts.
- **D-04:** All AI calls go through `/api/claude` route handler (Phase 4 infrastructure). Loading state shown on the generate button while request is in flight.
- **D-05:** AI generate buttons to restore: Brand Journey Statement, Avatar 1 Statement, Avatar 2 Statement, Ikigai Center, Origin Story (Brand Foundation); Font Identifier, Color Extraction from photo (Visual World); Lead Magnet Outline, Bio (Launch).

### Creator Analysis AI system
- **D-06:** Match old behavior exactly — dynamic "Add Creator" button, unlimited creators, Instagram handle input, AI analyze button per creator. Creator cards with profile, feed analysis, stats, and per-creator notes (multiple note fields per creator).
- **D-07:** Per-creator data stored as JSON array in blp_responses (e.g., `vw_ca_creators: [{handle, links, analysis, notes}]`). Auto-saved like other fields.

### Mood Board gallery
- **D-08:** Drag-and-drop image uploads into 4 categories (colorgrading, fonts, shots, colors) with mosaic gallery view. Matches old app exactly.
- **D-09:** Images stored in localStorage as base64 — same as old app. Accept the limitations (lost on browser clear, not cross-device).
- **D-10:** Pinterest embed preserved — auto-embed Pinterest board from user's link field using Pinterest embed script.

### Visual World Typography
- **D-11:** AI Font Identifier — upload image, AI identifies fonts via `/api/claude`. Matches old app.
- **D-12:** Font pairing suggestions — click mood card, get AI pairings, "Use This Pairing" auto-fill button. Matches old app.
- **D-13:** Google Fonts autocomplete suggestions on font input fields. Live font preview rendering. Italic/bold toggle buttons per font with hidden state fields (`vw_typo_primary_italic`, `vw_typo_primary_bold`, `vw_typo_body_italic`, `vw_typo_body_bold`).

### Visual World Color Palette
- **D-14:** Clickable color picker on swatches — click swatch to open color picker dialog. Matches old app.
- **D-15:** "Extract Colors From a Photo" — drag-and-drop image upload, AI extracts palette via `/api/claude`. Matches old app.

### Content module gaps
- **D-16:** Cinematic Content section (Workshop 10) — full instructional content covering camera settings, lighting, composition, color grading, shot types (with images), creative framing, camera movement, B-roll guidance. Educational only, no form fields. Add as new section in Content module.
- **D-17:** Content Blueprint/Playbook view — new section in Content module matching old app's structure. Includes 19 unique fields: 3 brand colors, mood vibe (option selector), narrative, objects, references, + 12 creator analysis fields (3 creators x 4 fields each: name, why, takeaway, difference). Plus compiled summary and 30-day Content Elevation Plan.
- **D-18:** Idea Generation angles — restore 80 individual angle fields (5 pillars x 4 ideas x 4 angles) replacing current 20 textarea approach. Matches old app's granular structure.

### Brand Foundation gaps
- **D-19:** Values Audit — 12 scored items across 4 categories (Values Check, Principles Check, Content Questions, Tough Situations) with chart visualization. Chart library at Claude's discretion (inline SVG consistent with ProgressRing pattern, or Recharts if complexity warrants it). Audit scores saved to blp_responses (`bf_audit_*` keys).

### Launch gaps
- **D-20:** "Set My Date" button on Goals section — auto-calculates 90 days from today and fills the review date field. Simple JS date calculation.

### Field naming convention
- **D-21:** Keep existing `bf_`/`vw_`/`ct_`/`la_` prefix convention for all new fields. Prefixes are already established across 26 components with live data.

### Claude's Discretion
- Chart library choice for Values Audit (inline SVG vs Recharts)
- Loading/error states for AI generation buttons
- Exact layout of creator analysis cards (follow old HTML as reference)
- Image upload component implementation details
- Google Fonts autocomplete implementation approach

</decisions>

<code_context>
## Existing Code Insights

### Reusable Assets
- `/api/claude` route handler: All AI generate buttons call this — already handles auth, size limits, edge function proxy
- `useAutoSave` hook: All new fields use this for persistence
- `WorkshopTextarea`, `WorkshopInput`, `OptionSelector`: Existing form components for new fields
- `ProgressRing`: Inline SVG pattern — can inform Values Audit chart approach

### Established Patterns
- `supabase as any` cast for blp_responses queries (Phase 5 convention)
- `setValue()` from react-hook-form for programmatic field updates (AI generation target)
- Section components load data on mount with `.from('blp_responses').select('responses').eq(...)` pattern
- `isGenerating` state key pattern for tracking per-button AI generation state (used in trust-and-money.tsx)

### Integration Points
- MODULE_SECTIONS config in `[section]/page.tsx` — new sections (Cinematic Content, Content Blueprint) need SECTION_REGISTRY entries
- `src/lib/modules.ts` — module section definitions need updating for new fields
- Content module overview — stats need updating if section count changes

</code_context>
