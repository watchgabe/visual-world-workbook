### Task 2 — Core Values UI with audit

Added a collapsible Values Audit section below the existing 6 core values inputs:

- **Toggle button** shows total score (`totalScore/40`) and a collapse arrow; starts closed
- **4 category cards** — Values Check, Principles Check, Content Questions, Tough Situations
- **12 questions** — 3 per category, brand-strategy questions written to match category spirit (old HTML had CSS/JS for audit but no HTML rendering the questions — known gap per research notes)
- **Score buttons** — 0/2/4 per item, selected state highlighted in `var(--orange)`, click calls `setValue(item.key, String(pts))`
- **Live score computation** — `vcScore`, `pcScore`, `cqScore`, `tsScore`, `totalScore` computed from `responses` (watch() cast as `Record<string, string>`)
- **Inline SVG bar chart** — 4 horizontal bars with `transition: 'width 0.3s ease'`, category label + `/10` score, width proportional to `Math.min(score/10, 1)`
- **Total score display** — large number with `/40` label + contextual feedback text
- **Auto-save** — `auditValues` array spread into `useEffect` deps triggers 500ms debounced upsert on any score change

### Implementation Notes

**Question text gap (known):** The old HTML had CSS (`.audit-cat`, `.audit-opt`, etc.) and JavaScript (`selectAudit`, `updateAuditScores`) for the audit, but no HTML markup rendering the questions themselves. The plan noted this as a known gap and directed writing 12 brand-strategy questions. All 12 were written to match the category names and spirit.

**Score options clarification:** Old JS shows `vc + '/10'` for category max but uses dynamic `pts` values. With 3 items at 0/2/4 each, the theoretical max is 12. The `/10` label is kept from the old app, bars are capped at 100% via `Math.min(score/10, 1)`.

**TypeScript pattern:** Used `responses = watch() as Record<string, string>` for dynamic key access to avoid union type narrowing errors with template-literal field keys — consistent with the `getValues() as Record<string, any>` cast used in avatar.tsx per D-19.

**Pre-existing build errors:** `npm run build` reports "Failed to compile" due to pre-existing unused variable errors in `src/app/(app)/modules/playbook/page.tsx`. These are out of scope — `npx tsc --noEmit` passes with no errors in core-values.tsx or modules.ts.
