# Phase 05.1 Plan 01: AI Generate Buttons + Set My Date Summary

**One-liner:** Added 7 AI generate buttons (brand journey statement, avatar 1+2, ikigai center, origin story, polished bio, lead magnet outline) and a 90-day date calculator, all with verbatim prompts from the old HTML app.

## What Was Built

All 8 augmentations from the plan were added to 7 existing component files. No new files were created.

### Brand Foundation (4 files, 5 buttons)

**brand-journey.tsx** — Added `isGenerating` state and `handleGenerateJourney()`. Generate button appears above `bf_journey_statement` textarea. Prompt verbatim from old `generateBrandJourney()`: fills the "I want to [X] by being known for [Y] through [Z] starting with [W]" template. Guard: all 4 fields must be non-empty.

**avatar.tsx** — Added `isGenerating` state and `handleGenerateAvatar(num: 1 | 2)`. Two generate buttons — one above `bf_av1_statement`, one above `bf_av2_statement`. Prompt verbatim from old `generateAvatar(num)`: extracts NAME: line, strips it, populates statement field with the paragraph. Guard: situation and struggle must be non-empty.

**core-mission.tsx** — Added `isGenerating` state and `handleGenerateIkigai()`. Generate button above `bf_ikigai_center` textarea. Prompt verbatim from old `generateIkigai()`: 3-sentence format identifying the thread through all four Ikigai quadrants. Guard: all 4 fields must be non-empty.

**origin-story.tsx** — Added `isGenerating` state and `handleGenerateOriginStory()`. Generate button above `bf_origin_story` textarea. Prompt verbatim from old `generateOriginStory()`: strict "use only what's written, no assumptions" rules with Part 1–4 structure. Guard: story1 and story2 must be non-empty.

### Launch (3 files, 3 buttons)

**bio.tsx** — Added `isGenerating` state, `bioResult` state, and `handleGenerateBio()`. Generate button appears after bio preview. AI response shown in read-only div with `white-space: pre-wrap` and orange section label (matches old `showResult()` pattern). Cross-section fields `la_funnel_cta` and `la_lm_name` included in `defaultValues` and loaded from blp_responses upsert. Prompt includes all 6 fields per D-03.

**lead-magnet.tsx** — Added `isGenerating` state and `handleGenerateLeadMagnet()`. Generate button above `la_lm_outline` textarea. `setValue` populates the field, triggering auto-save. Prompt verbatim from old `generateLeadMagnet()`.

**goals.tsx** — Added `setDate()` function. "Set My Date (90 Days)" button placed inline next to `la_goal_review_date` input in a flex row. Pure client-side date calculation: `new Date()` + 90 days, formatted as "Month Day, Year" using months array verbatim from old `setReviewDate()`.

## Deviations from Plan

None - plan executed exactly as written.

The TypeScript errors encountered were handled via the same `getValues() as Record<string, any>` pattern already established in the codebase (consistent with existing decisions in STATE.md for dynamic field access).

## Known Stubs

None — all 7 AI generate buttons are wired to live `/api/claude` calls. The `bioResult` state initializes empty and shows nothing until generated.
