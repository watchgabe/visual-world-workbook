## Nyquist Compliance Rationale

This phase remediates UI features that inherently require browser interaction to verify: drag-and-drop image uploads, AI API calls with visual results, chart rendering, color pickers, font previews, and Pinterest embeds. Automated behavioral tests would require a full E2E framework (Playwright/Cypress) which is out of scope for this remediation phase and would consume significant context for setup alone.

**Verification approach:**
- **Structural checks (automated):** TypeScript compilation (`npx tsc --noEmit`) and `npm run build` after every task confirm code correctness, import resolution, and type safety.
- **Content checks (automated):** `grep` commands in `<verify>` blocks confirm key patterns exist (field names, API calls, state management).
- **Behavioral checks (manual):** Feature walkthrough after each plan wave confirms UI behavior matches old app 1:1.

This combination provides sufficient coverage for a UI-feature remediation phase. Wave 0 test scaffolding is not needed -- the structural checks plus manual walkthrough are the appropriate verification strategy.
