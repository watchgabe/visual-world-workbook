# Phase 05.1 Plan 06: Typography AI Features Summary

AI font identification from uploaded image, mood-card font pairing suggestions with static library, Google Fonts live preview with debounced link injection, autocomplete for font name inputs, and italic/bold toggle buttons with hidden state fields.

## What Was Built

### Task 1 — modules.ts: 4 new italic/bold toggle fields
Added `vw_typo_primary_italic`, `vw_typo_primary_bold`, `vw_typo_body_italic`, `vw_typo_body_bold` to the typography section in `MODULE_SECTIONS['visual-world']`. All have `required: false`. These store `"true"` or `""` and auto-save via the existing `useAutoSave` pattern like any other field.

### Task 2 — typography.tsx: 4 interactive systems added

**1. AI Font Identifier (per D-11)**
- Drag-and-drop / click-to-upload image zone with thumbnail preview
- "Identify Fonts" button sends base64 image to `/api/claude` with a prompt requesting 3 Google Fonts with role/description/isItalic
- Results render as a list with font name, role badge, description, and preview text in the identified font
- Each result has "Use as Primary/Body" button; a "Use Headline + Body Suggestion" button auto-fills both fields at once

**2. Font Pairing Suggestions (per D-12)**
- 6 mood cards (Dark & Cinematic, Clean & Minimal, Bold & Statement, Warm & Approachable, Luxury & Editorial, Creative & Expressive)
- 36 curated pairings migrated 1:1 from old app's TYP_PAIRINGS static table
- Click mood card to see first 3 pairings with typography preview rendered using the actual Google Fonts
- "See 3 More Options" cycles through remaining pairings; "Use This Pairing" fills both font fields via setValue

**3. Google Fonts Live Preview (per D-13)**
- `useEffect` watches `vw_typo_primary` and `vw_typo_body` with 500ms debounce
- On change, injects `<link>` tag with Google Fonts URL (id `gf-primary` / `gf-body` to prevent duplicates)
- Preview block shows regular, italic, and bold variants of "The quick brown fox..."
- Preview text style reflects current italic/bold toggle state

**4. Font Autocomplete + Italic/Bold Toggles (per D-13)**
- `FontAutocomplete` inline component: static array of 80 popular Google Fonts, filters by prefix, dropdown closes on outside click or selection
- Hidden `WorkshopInput` elements for `vw_typo_primary`/`vw_typo_body` preserve auto-save trigger pattern
- I/B toggle buttons per font: orange background when active, writes to hidden italic/bold fields via `toggleStyle()`, preview updates immediately

## Decisions Made

1. **Static pairing library, not AI calls**: The old app's Font Finder used a hardcoded `TYP_PAIRINGS` table, not Claude AI. The plan suggested AI but the old app's actual behavior was static. Migrated the full table (36 pairings) 1:1. AI is only used for font identification from images.

2. **FontAutocomplete as inline component**: Kept within typography.tsx since it is only needed there. No other section has font-name inputs. Avoids premature extraction to a shared component.

3. **Hidden WorkshopInput for auto-save**: The custom `FontAutocomplete` replaces the visible input but the auto-save hook watches `WorkshopInput`'s debounced changes. Hidden `WorkshopInput` elements preserve this without duplicating the debounce logic.

4. **runFontIdentification() naming**: Renamed from `identifyFonts()` to avoid TypeScript duplicate identifier error with the `fontSuggestions` state array (originally named `identifyFonts`).
