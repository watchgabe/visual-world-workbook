    - Tailwind: match old admin `.config-strip` pattern — `bg-[var(--surface)] border border-[var(--border2)] rounded-md p-4 mb-6 flex flex-wrap gap-3 items-end`

    - 4 cards in a grid: Total Users, Started (has any module data), Completed All 4 (all 4 modules have data), Active Last 7 Days (last_sign_in_at within 7 days)
    - Compute "started" and "completed" by checking if responsesByUser[userId] has non-empty responses for each of brand-foundation, visual-world, content, launch
    - A user "has data" for a module if the responses object has any non-empty string values (excluding SKIP_KEYS)
    - Import SKIP_KEYS from `@/lib/admin/labels`
    - Stat number uses `font-family: 'Barlow Condensed'` with `text-4xl font-black` (matching old admin `.stat-num`)
    - Grid: `grid grid-cols-4 gap-3 mb-8` (responsive: `grid-cols-2 sm:grid-cols-4`)

    **User Table**:
    - Table with columns: avatar, name, email, module badges, last seen, expand chevron
    - Avatar: first letter initial in orange-tinted circle (matching old admin `.u-avatar`)
    - Module badges: 4 small pills "01 Foundation", "02 Visuals", "03 Content", "04 Launch" — green if has data, gray if not (matching old admin `.mbadge.has`)
    - Last seen: `relTime()` helper — port from old admin (lines 753-763): returns "just now", "Xm ago", "Xh ago", "Xd ago", or date string
    - Click row toggles `expandedUserId` state (only one expanded at a time, per D-03)
    - Expanded row renders `<UserDetailPanel>` inline in a full-width colspan cell below the user row

    **2. Create src/components/admin/UserDetailPanel.tsx** (Client Component):
    - 'use client' directive
    - Props: `{ user: User, userResponses: Record<string, Record<string, unknown>>, circleApiKey: string, circleCommunityId: string, onDeleted: () => void }`

    **Progress Summary Bar** (matching old admin `.prog-summary`):
    - Left: avatar + name + email
    - Divider
    - Right: Overall progress bar + 4 module progress bars
    - Progress calculation: for each module, count non-empty string fields in responses (excluding SKIP_KEYS) as a fraction of total MODULE_SECTIONS field count for that module
    - Import MODULE_SECTIONS from `@/lib/modules` to get field counts
    - Overall = sum of all module done sections / total sections
    - Progress bar fill: orange normally, green (`var(--green-text)`) when 100%

    **Delete Button** (per D-07):
    - Red-tinted button "Delete User" at top-right of detail panel
    - On click: show confirmation via `useState` inline dialog (not window.confirm) — display: "Delete all data for {email}? This removes their account and all saved progress. This cannot be undone." with Cancel and Delete buttons
    - On confirm: `fetch('/api/admin/delete-user', { method: 'POST', body: JSON.stringify({ userId: user.id }) })`
    - While deleting: button shows "Deleting..." and is disabled
    - On success: call `onDeleted()` which triggers `router.refresh()` in parent to re-fetch server component data

    **Circle.so Integration** (per D-06):
    - If circleApiKey and circleCommunityId are set, fetch Circle member on panel mount:
      `fetch('/api/circle', { method: 'POST', body: JSON.stringify({ action: 'get_member', email: user.email, community_id: circleCommunityId }) })`
    - If member found, render Circle card (matching old admin `.circle-card`): avatar, name, "Circle Member" badge, role, joined date, last active, "View in Circle" link
    - If not found: italic dim text "Not found in Circle community."
    - Badge Award button: below Circle card, button "Award Badge" that calls:
      `fetch('/api/circle', { method: 'POST', body: JSON.stringify({ action: 'award_badge', email: user.email, community_id: circleCommunityId }) })`
    - Show success/error feedback after badge call

    **Answer Grid** (per D-04 — 1:1 migration):
    - 4 module sections, each with header (module number + title + field count)
    - For each module, iterate responsesByUser entries for that module_slug
    - Map module_slug to module data: 'brand-foundation', 'visual-world', 'content', 'launch'
    - For each field in responses:
      - Skip keys in SKIP_KEYS
      - Use `getLabelForKey(key)` from labels.ts to get human-readable label
      - Separate option fields (keys starting with `opt_`) into chip display
      - Text fields render as card with label + value
    - Match old admin's `.ans-grid` pattern: grid of cards with label in small dim text and value below
    - Match old admin's `.opt-chip` pattern for option selections
    - Empty module: show dim italic "Sections marked complete but no text answers saved yet"
    - For content module fields: use isCCKey() to correctly categorize fields when needed

    **Responsive**: entire dashboard max-width 1200px centered, padding 2rem. Table wraps on mobile. Stat cards go 2-col on small screens.
