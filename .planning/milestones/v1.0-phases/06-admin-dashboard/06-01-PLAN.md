    **2. Create src/app/api/admin/delete-user/route.ts** (per D-07):
    - Import `createClient` from `@/lib/supabase/server` (for auth check) and `createServiceClient` from `@/lib/admin/service-client` (for delete)
    - POST handler:
      1. Auth guard: `const supabase = await createClient(); const { data: { user } } = await supabase.auth.getUser()`. If no user, return 401.
      2. Admin check: if `user.app_metadata?.role !== 'admin'`, return 403 `{ error: 'Forbidden' }`.
      3. Parse `{ userId }` from request body. If missing, return 400 `{ error: 'userId required' }`.
      4. `const service = createServiceClient(); const { error } = await service.auth.admin.deleteUser(userId)`. blp_responses cascade-deletes via FK.
      5. If error, return 500 `{ error: error.message }`. Otherwise return `{ ok: true }`.

    **3. Create src/app/api/admin/circle-config/route.ts** (per D-05):
    - Import `createClient` from `@/lib/supabase/server` and `createServiceClient` from `@/lib/admin/service-client`
    - Both GET and POST require admin auth (same pattern as delete-user: getUser + app_metadata.role check)
    - GET handler: reads from blp_config where key in ('circle_api_key', 'circle_community_id'). Returns `{ circle_api_key: string | null, circle_community_id: string | null }`.
    - POST handler: accepts `{ circle_api_key?: string, circle_community_id?: string }`. Upserts each provided key into blp_config using `service.from('blp_config').upsert({ key, value })`. Returns `{ ok: true }`.
