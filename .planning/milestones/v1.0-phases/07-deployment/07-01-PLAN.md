The user must complete these manual steps (Claude cannot authenticate to Vercel or Supabase dashboards):

**Step A — Create Vercel project (per D-01, D-02):**
1. Go to https://vercel.com/dashboard
2. Click "Add New" > "Project"
3. Import the Git repository for this project
4. Vercel will auto-detect Next.js — accept the defaults
5. Before clicking "Deploy", add the 3 environment variables (Step B)

**Step B — Set environment variables in Vercel (per D-03, D-04):**
Add these in the Vercel project settings (or during initial project creation):

| Variable | Value Source | Scope |
|----------|-------------|-------|
| `NEXT_PUBLIC_SUPABASE_URL` | Supabase Dashboard > Settings > API > Project URL | Production, Preview, Development |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | Supabase Dashboard > Settings > API > anon public | Production, Preview, Development |
| `SUPABASE_SERVICE_ROLE_KEY` | Supabase Dashboard > Settings > API > service_role | Production, Preview, Development |

6. Click "Deploy"

**Step C — Add production redirect URL in Supabase (per D-06, D-07):**
1. After Vercel deploys, copy the production URL (e.g., `https://your-app.vercel.app`)
2. Go to Supabase Dashboard > Authentication > URL Configuration
3. Under "Redirect URLs", click "Add URL"
4. Add: `https://your-app.vercel.app/auth/callback`
5. Also verify that `http://localhost:3000/auth/callback` is already in the list (for local dev)
6. Save

**Step D — Verify deployment (per D-05, replaces D-08 smoke test):**
1. Visit the production URL — the login page should load
2. View page source or open DevTools Network tab — confirm no Supabase keys appear in the HTML or client JS bundle (only `NEXT_PUBLIC_` prefixed values should be visible, which is expected and safe)
3. Send yourself a magic link — confirm the email contains the production URL (not localhost)
4. Click the magic link — confirm you land on the dashboard as a signed-in user
5. Navigate to a module, type in a form field, verify auto-save works
6. Sign out and confirm redirect to login page
